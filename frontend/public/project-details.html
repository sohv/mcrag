Project Overview: Multi Code Review and Generation (MCRAG)

  • GAN-Inspired Design: The system is conceptually inspired by Generative Adversarial Networks (GANs), featuring a collaborative dynamic between a generator and critics.
  • Two-Part System:
    • Generator: Produces code based on user prompts.
    • Critics: Review the generated code, providing detailed feedback and suggestions for improvement.
  • Iterative Refinement: The generator incorporates critic feedback to refine its code. This process is repeated for three iterations:
    Generate → Review → Refine
    This loop helps minimize errors and bugs in the final code.
  • Multi-Critic Setup: For higher code quality, the system uses one generator and two critics:
    • Generator: Gemini 2.5-Flash
    • Critic 1: GPT-4o Mini
    • Critic 2: DeepSeek R1
  • Feedback Ranking: After each review, the generator evaluates and ranks the feedback from both critics, prioritizing the most valuable suggestions for incorporation. This ensures that the most impactful improvements are made in each refinement cycle.
  • Outcome: After three cycles, the system outputs code that has undergone multiple rounds of expert review and refinement, resulting in robust, high-quality code with minimal chances of errors or bugs.

Example Workflow

Suppose a user submits the following prompt:

"Write a Python function that checks if a number is prime."
  1. Generation: The generator (Gemini 2.5-Flash) creates an initial Python function for primality checking.
  2. Review: Two critics (GPT-4o Mini and DeepSeek R1) independently review the code. For example:
    • Critic 1 points out that the function does not handle numbers less than 2 and could be optimized for even numbers.
    • Critic 2 suggests using the math.isqrt function for efficiency and recommends adding type hints.
  3. Ranking & Refinement: The generator ranks the feedback, prioritizes the most valuable suggestions (e.g., input validation and efficiency improvements), and refines the code accordingly.
  4. Iteration: Steps 2 and 3 are repeated for a total of three cycles. With each iteration, the code becomes more robust, readable, and efficient, as new issues are caught and improvements are incorporated.
  5. Final Output: After three iterations, the user receives a well-tested, high-quality Python function with minimal chances of errors or bugs.

This workflow ensures that the generated code is not only functional but also reviewed and refined by multiple expert perspectives, closely mirroring the adversarial and collaborative spirit of GANs.
